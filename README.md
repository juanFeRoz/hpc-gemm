# Matrix Multiplication Autotuning & Statistical Analysis

This repository contains an autotuning and benchmarking framework for matrix-matrix multiplication kernels implemented for SYCL and CUDA. It compiles parameterized kernels, runs autotuning/search strategies, and includes scripts for running statistical experiments and analyzing results.

## Project Structure

- `autotune.py` — autotuning utilities and search strategies (Bayesian, Random).
- `run_statistical_experiments.py` — run batched experiments and record results to JSON.
- `statistical_analysis.py` — analyze experiment output and generate summary plots/metrics.
- `wrapper.py` — ctypes wrapper to call compiled shared libraries from Python.
- `kernel_matmul_sycl.cpp`, `kernel_matmul_cuda.cu` — matmul kernel sources for SYCL / CUDA.
- `Makefile` — build convenience rules for compiling kernels to shared libraries.
- `statistical_results.json`, `convergence_analysis.json` — example output produced by the analysis pipeline.

## Requirements

- Python 3.8+
- optuna (for Bayesian tuning, optional)
- numpy, matplotlib
- A working SYCL toolchain (oneAPI or equivalent) for SYCL builds
- CUDA toolchain (`nvcc`) for CUDA builds

Install Python deps:

```bash
python3 -m pip install optuna numpy matplotlib
```

## Quickstart

1. Run a set of statistical/autotuning experiments and record results:

```bash
python3 run_statistical_experiments.py
```

2. Analyze the generated results and produce plots:

```bash
python3 statistical_analysis.py statistical_results.json
```

Typical outputs produced by the analysis pipeline:
- `statistical_results.json` — raw numeric results and metadata
- `convergence_analysis.json` — convergence summaries across trials
- PNG plots (comparison and summary figures) generated by `statistical_analysis.py`

## Autotuning

Autotuning logic resides in `autotune.py`. Available strategies include:

- `bayesian_search(...)` — Optuna (recommended when trials are limited)
- `random_search(...)` — random sampling baseline

The autotuner supports `matmul` kernels. Configure search-space parameters in `autotune.py`.

Example search-space parameters (customize as needed):
- `BM` (block M) ∈ {32, 64, 128}
- `BN` (block N) ∈ {32, 64, 128}
- `BK` (block K) ∈ {8, 16, 32}
- `TM` (thread multiplier M) ∈ {1, 2, 4}
- `TN` (thread multiplier N) ∈ {1, 2, 4}

Constraints are enforced by the helper `is_valid_config` (e.g., threads-per-block limits).

## Building kernels manually

Use the Makefile to compile parameterized shared libraries (adjust flags for your toolchain):

```bash
make kernel_matmul_sycl.so BM=32 BN=32 BK=32 TM=4 TN=4
make kernel_matmul_cuda.so BM=32 BN=32 BK=32 TM=4 TN=4
```
